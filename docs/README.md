# pi-ws Documentation

This directory contains human-written guides and generated API documentation for
`pi-ws`.

## Start Here

- [Getting started](GETTING_STARTED.md) - install the package, start a local
  bridge, and connect a WebSocket client.
- [Examples](../examples/README.md) - run the browser chat demo and configure
  provider keys, auth, artifacts, and sandbox paths.
- [Security guide](SECURITY.md) - understand what `pi-ws` protects by default,
  what remains your application's responsibility, and how to deploy it safely.
- [Contributing guide](CONTRIBUTING.md) - set up the repository, run checks,
  update generated docs, and prepare pull requests.
- [Superpowers](superpowers/) - optional local Pi/Codex integration notes.
- [Changelog](CHANGELOG.md) - released changes.
- [API reference](api/index.md) - generated TypeScript API docs.

## Documentation Types

- Guides are maintained by hand and should explain user intent, workflows, and
  operational tradeoffs.
- API reference files under `docs/api/` are generated by `pnpm build:docs`.
  Do not edit generated API files directly.
- Example docs live next to runnable example code in `examples/`.
